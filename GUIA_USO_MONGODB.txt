🚀 CÓMO USAR MONGODB
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

OPCIÓN 1: Usar el Script Automático (RECOMENDADO)
-------------------------------------------------

   1. Ejecutar script de cambio:

      .\cambiar-mongodb.ps1

   2. Ejecutar la aplicación:

      .\mvnw.cmd javafx:run

   3. Seleccionar "MongoDB" cuando aparezca el diálogo

   4. ¡Listo! La aplicación se conectará a MongoDB


OPCIÓN 2: Configuración Manual
------------------------------

   1. Editar database.properties:

      db.type=mongodb
      db.host=localhost
      db.port=27017
      db.name=taller_db
      db.user=
      db.password=

   2. Ejecutar la aplicación:

      .\mvnw.cmd javafx:run

   3. Seleccionar "MongoDB"

💡 NOTAS IMPORTANTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. MongoDB debe estar ejecutándose antes de iniciar la aplicación

   Verificar con:
   mongosh --version

2. Por defecto MongoDB NO requiere autenticación en localhost

   Si tu instalación requiere usuario/contraseña, configúralo en
   database.properties

3. La aplicación detecta automáticamente el tipo de base de datos

   Puedes cambiar db.type en database.properties sin modificar código

4. Los scripts de cambio crean backup automático

   Se guarda en database.properties.backup

🛠️ SOLUCIÓN DE PROBLEMAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROBLEMA: "No se pudo conectar a MongoDB"
SOLUCIÓN:
   1. Verificar que MongoDB esté ejecutándose
   2. Verificar host y puerto en database.properties
   3. Si usa autenticación, verificar usuario/contraseña

PROBLEMA: "Base de datos taller_db no existe"
SOLUCIÓN:
   Importar los datos con: cd mongodb; .\importar-mongodb.ps1
   O crear la base manualmente en MongoDB

PROBLEMA: "Error de autenticación"
SOLUCIÓN:
   1. Si NO usas autenticación, dejar user y password vacíos
   2. Si usas autenticación, verificar credenciales correctas
   3. Verificar que el usuario tenga permisos en la BD

PROBLEMA: "Driver MongoDB no encontrado"
SOLUCIÓN:
   Ejecutar: .\mvnw.cmd clean compile
   Maven descargará automáticamente las dependencias
